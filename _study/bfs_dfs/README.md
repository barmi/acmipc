# DFS (Depth-First Search) & BFS (Breadth-First Search) 학습 가이드

이 디렉토리는 그래프 탐색의 핵심 알고리즘인 DFS와 BFS를 학습하기 위한 자료와 예제 코드를 포함하고 있습니다.

## 1. 기본 개념

### 🔍 DFS (깊이 우선 탐색, Depth-First Search)
- **정의**: 갈 수 있는 만큼 최대한 깊게 가보고, 더 이상 갈 곳이 없으면 이전 단계로 돌아와 다른 길을 찾는 방식입니다.
- **특징**:
  - **스택(Stack)** 또는 **재귀(Recursion)** 함수를 사용하여 구현합니다.
  - 한 방향으로 끝까지 파고드는 성질이 있어, 모든 노드를 방문하고자 할 때 주로 사용합니다.
- **주요 소스**: [dfs_bfs_basic.cpp](./dfs_bfs_basic.cpp) (DFS 구현 포함)

### 🌐 BFS (너비 우선 탐색, Breadth-First Search)
- **정의**: 시작 노드에서 가까운 노드부터 차례대로 방문하는 방식입니다.
- **특징**:
  - **큐(Queue)** 자료구조를 사용하여 구현합니다.
  - 층별(Level-order)로 탐색하며, 시작점으로부터의 거리가 짧은 노드부터 방문합니다.
- **주요 소스**: [dfs_bfs_basic.cpp](./dfs_bfs_basic.cpp) (BFS 구현 포함)

---

## 2. DFS vs BFS 비교

| 특징 | DFS (깊이 우선 탐색) | BFS (너비 우선 탐색) |
| :--- | :--- | :--- |
| **자료구조** | 재귀 또는 스택 | 큐 (Queue) |
| **최단 거리** | 보장되지 않음 | **최단 거리 보장** (가중치 없는 그래프) |
| **공간 복잡도** | 상대적으로 적은 메모리 사용 | 노드가 많을수록 큐에 저장할 데이터 급증 |
| **주요 용도** | 모든 노드 방문, 경로 특징 저장, 사이클 탐색 | 최단 경로 찾기, 네트워크 탐색, 최소 횟수 계산 |

## 5. 추가 팁

- **시간 복잡도**: $O(V + E)$ (V: 정점 수, E: 간선 수)
- **방문 체크**: 무한 루프 방지를 위해 `visited` 배열 관리가 필수입니다.
- **스택 오버플로우**: DFS 재귀 구현 시 깊이가 너무 깊어지면 주의가 필요합니다.
